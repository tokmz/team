# 项目记忆协议

## 目的

从项目实战中提取「决策 + 原因」和「踩坑 + 教训」，供后续会话直接复用，避免重复踩坑。

## 记忆文件位置

```
{项目根目录}/.claude/memory/team-project.md
```

## 记什么

| 类型 | 格式 | 示例 |
|------|------|------|
| 架构决策 | `决策: 为什么选X不选Y → 原因` | `ORM选ent不选GORM → 团队更熟悉type-safe查询` |
| 踩坑记录 | `踩坑: 什么问题 → 根因 → 解法` | `Alpine镜像build失败 → 缺CGO依赖 → 换golang:1.21镜像` |
| 团队约定 | `约定: 规则 → 原因` | `错误码从10000开始 → 预留系统错误段` |
| 性能决策 | `性能: 决策 → 数据` | `用户表按tenant_id分库 → 单表超500万行，查询P99从2s降到50ms` |
| 安全教训 | `安全: 风险 → 修复` | `接口越权 → 加了tenant_id鉴权中间件` |

## 不记什么

- 文件路径（会变）
- 代码内容（从文件读）
- 库版本号（从 go.mod 读）
- 可从代码推导的事实
- 超过 30 天的临时决策

## 写入时机

1. **项目阶段完成时** — 架构设计完、数据库设计完、模块开发完
2. **踩坑修复后** — bug 修完、部署失败后解决
3. **用户明确说"记一下"时**
4. **项目交付时** — 最终总结

## 写入格式

```markdown
# 项目记忆 — {项目名}

> 最后更新: YYYY-MM-DD
> 记忆条数: N/10

## 决策
- [日期] {决策内容} → {原因}
- [日期] {决策内容} → {原因}

## 踩坑
- [日期] {问题} → {根因} → {解法}

## 约定
- [日期] {约定内容} → {原因}

## 性能
- [日期] {决策} → {数据证据}
```

## 硬规则

1. **上限 10 条** — 超过 10 条时，删除最旧的或最不重要的
2. **只记 WHY** — 不记 WHAT（WHAT 从代码读，不会错）
3. **带日期** — 每条记忆必须有日期，超过 30 天标记 `⚠️可能过期`
4. **使用前验证** — 读取记忆后，先验证对应的代码是否还存在，避免幻觉
5. **一个项目一个文件** — 不跨项目混记忆

## 读取时机

1. **新会话启动时** — 如果项目目录有 `.claude/memory/team-project.md`，自动加载
2. **新功能开发前** — 读记忆了解已有决策和踩坑
3. **维护模式** — 读记忆快速了解项目历史
